# KOSIS 학습 로그 (LEARNINGS)

> **AI 진화 노트.** KOSIS를 쓰며 겪은 **실패·성공·함정**을 누적한다.
> 이 파일은 KOSIS 스킬의 "오답노트"다. 매 작업 전에 읽고, 새로 배운 것을 매 작업 후에 적어 스스로 진화한다.

## 사용 규칙 (AI 필독)

1. **조회 전 READ**: KOSIS 작업을 시작하면 이 파일을 먼저 읽는다. 특히 `## 테이블별 함정`과 `## 오류코드 대응`을 확인해 같은 실수를 반복하지 않는다.
2. **작업 후 APPEND**: 주목할 만한 **실패**(예상 못한 오류·빈 결과)나 **성공**(처음 통한 쿼리·해결한 함정)을 겪으면 아래 해당 섹션에 **한 줄 이상** 추가한다.
3. **형식 유지**: 각 항목은 `날짜 | 테이블/명령 | 무슨 일 | 왜 | 다음엔 이렇게` 5요소를 가능한 한 채운다. "왜"가 가장 중요하다.
4. **중복 금지·승격**: 이미 있는 항목이면 새로 쓰지 말고 기존 항목을 보강한다. 같은 함정이 3회 이상 반복되면 `## 일반 규칙`으로 승격한다.
5. **검증된 것만 성공으로**: 실제 실행해 데이터가 온전히 나온 쿼리만 `## 성공 쿼리 패턴`에 올린다(추측 금지).

## 일반 규칙 (반복 검증된 핵심)

- **조회 전 `kosis m <ORG> <TBL>` 필수**: 통계표마다 분류축(objL) 개수와 코드 형식이 다르다. 메타 없이 `-c1 ALL`만 넣으면 `오류 [20] objL 누락`이 난다.
- **분류축이 2개 이상이면 `-c1`뿐 아니라 `-c2`(…)도 채워야 한다**: 메타의 `분류ID`가 2개(예: A 시군구 + YRE 연령)면 `-c1 ALL -c2 ALL` 모두 필요.
- **자동 분할은 기간(시점) 축으로만 일어난다**: 4만 셀 초과 시 시점을 쪼개 병렬 조회한다. **단, 단일 시점 하나가 이미 4만 셀을 넘으면 분할로도 실패**한다(아래 함정 참조).
- **병렬은 키 2개 이상일 때만 발동**: 키 1개면 순차. 큰 조회가 잦으면 `kosis config add-key`로 키를 늘리면 빨라진다(실측 키3개 ≈ 키1개의 2.2배).
- **기계 결합엔 코드, 표시엔 이름**: 지도(choropleth)·DB join이 목적이면 `C1`/`ITM_ID` 같은 **코드 키**를 먼저 확보하라(`-f json`/`--with-code`). 코드가 `null`로 비면 `meta -f json`의 CLASSIFICATIONS로 역매핑한다. 한글 이름은 중복(예 '중구')이라 join 키로 부적합. 상세 절차는 `references/16-geo-join.md`. (제조업 출하액뿐 아니라 모든 지역·분류 결합의 공통 규칙)
- **일상어 ≠ KOSIS 항목명**: 검색이 0건/오답이면 통계 **조사명**(예 "광업제조업조사")으로 재검색하고 `meta` 항목명으로 동의어를 교정하라. "제조업 생산액"은 생산'지수'만 나오기 쉬움 — 시군구 실액수는 출하액(T04)/생산액(T07). 상세는 `references/15-ai-workflow.md` 용어 매핑 표.

## 오류코드 대응 (실측 보강)

| 코드 | 의미 | 즉효 대응 |
|------|------|-----------|
| `20` | 필수 변수 누락 (objL) | 분류축 개수를 `meta`로 확인 → `-c1`~`-cN` 모두 채우기 |
| `21` | 잘못된 요청 변수 | `meta`로 분류/항목 코드 재확인 |
| `30` | 데이터 없음 | 시점 확인, `-l 5`로 최근부터 |
| `31` | **40,000셀 초과** | **자동 분할로도 단일 시점이 한도 초과면** 분류(`-c1`/`-c2`)·항목(`-i`) 범위를 좁혀라. "전체×전체×다년도" 회피 |
| HTTP 429 | 분당 호출 제한 | 자동 백오프 재시도. 잦으면 키 추가(`config add-key`) |

## 성능 팁 (실측)

- 키 N개 = 병렬 워커 N개. **키 3개 조회가 키 1개보다 약 2.2배 빠름**(10청크 기준 3.12s vs 6.80s). 대용량을 자주 받으면 키를 늘려라.
- **`--concurrency N`(v0.3.2+)**: 동시 조회 워커 수를 직접 지정. 키가 1개여도 N개 동시 요청이 가능(워커-키 분리). 기본값은 자동(max(키 개수, 2)). 단 너무 높이면 429 빈발 가능.
- **와이드 테이블 2차 분할은 느리다**: 단일 시점이 4만 셀 초과면 분류 축으로 수십 그룹 분할되어 그룹마다 메타+조회가 일어남. 속도가 중요하면 분류/항목을 좁혀라.
- 동일 쿼리 재조회는 캐시 적중(기본 TTL 48h)으로 즉시. 캐시는 `~/.kosis/cache`.
