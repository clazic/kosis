# 10. 자동 분할 조회 (4만 셀 초과)

`ALL`로 전체 분류/항목을 선택하면 4만 셀을 초과할 수 있습니다. 이 경우 자동으로 시점 축 분할 조회를 수행합니다.

```bash
kosis d 101 DT_1IN1502 -c1 ALL -i ALL -p Y -s 2015 -e 2024 -o 인구.db
# ⚠ 예상 셀 수: ~796,800 (자동 분할 조회)
# [1/20] 2015년 조회 중...
# ...
# ✓ 인구.db 저장 완료
```

## 동작 방식

1. Meta API로 예상 셀 수 계산 (분류 × 항목 × 시점)
2. 4만 이하 → 일반 조회
3. 4만 초과 → **시점(기간) 축**으로 청크 분할 (Y/M/Q/H 단위)
4. 작업 큐 + 키 풀 기반 **병렬 실행** (워커 수는 키 개수와 분리, `--concurrency`로 제어)
5. 청크별로 키 풀에서 키를 빌려 요청 → 429 시 해당 키를 쿨다운하고 청크를 다른 가용 키로 재투입
6. **단일 시점이 4만 셀을 초과**하면(기간 분할로도 불가) → **분류 축으로 2차 분할** (아래 참조)

## 동시성 제어 (`--concurrency`)

동시 조회 워커 수를 직접 지정합니다. **키 개수와 무관**하게 동시 요청 수를 제어합니다.

```bash
# 동시 4개 요청 (키가 1개여도 병렬)
kosis d 101 DT_1IN1502 -c1 ALL -i ALL -p Y -s 2015 -e 2024 --concurrency 4 -o 인구.db

# 순차 강제 (1개씩)
kosis d ... --concurrency 1
```

- 미지정/0 → 자동: `max(키 개수, 2)`
- 너무 높이면 429(rate limit)가 빈발할 수 있음 → 키를 늘리거나 적정값 사용
- 키를 늘리면 429 분산 + 속도 향상:
  ```bash
  kosis config add-key <KEY2>
  kosis config key-list     # 등록된 키 확인
  ```

## 분류 축 2차 분할 (와이드 테이블)

시군구 × 1세별 × 성별처럼 **단일 시점 하나가 이미 4만 셀을 초과**하는 통계표는 기간 분할로 조회할 수 없습니다. 이 경우 가장 큰 분류 축(예: 시군구)을 코드 그룹으로 **자동 2차 분할**하여 조회 후 병합합니다.

```bash
# 단일 연도도 134만 셀인 와이드 테이블 → 분류 축으로 자동 2차 분할
kosis d 101 DT_1B04006 -c1 ALL -c2 ALL -i ALL -p Y -s 2023 -e 2023 -o pop.csv
# ℹ 단일 시점이 40000셀 제한을 초과하여 분류 축으로 N개 그룹으로 2차 분할합니다.
```

> 2차 분할은 그룹 수가 많아 **느릴 수 있습니다**. 속도가 중요하면 `--class1`/`--class2`/`--item`으로 범위를 좁히세요. 2차 분할조차 불가능하면 원인+해결 안내 메시지가 출력됩니다.

## 빈 구간 처리

데이터가 없는 시점 구간(API 오류 30)은 자동으로 건너뜁니다. 통계표 개편 등으로 특정 연도에 데이터가 없는 정상 상황입니다.

## 자동 분할 비활성화

```bash
# 직접 범위를 관리하고 싶을 때
kosis d 101 DT_1IN1502 -c1 "00" -i ALL -p Y -s 2020 -e 2024 --no-auto-split
```
